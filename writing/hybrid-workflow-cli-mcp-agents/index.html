The best AI workflow is hybrid

The workflow that works best for me is not CLI versus MCP versus agents. It is knowing what each one is good at and composing them intentionally.

March 18, 2026

5 min read

On this page

Back to writing

Most of the conversation around AI tooling turns into a fight between formats. People ask whether the command line is better than structured tool systems, whether agents are overhyped, or whether MCP is just extra ceremony.

In practice, that framing has not been useful for me. The better question is which interaction model gives me the right balance of speed, reliability, and leverage for the task in front of me.

The CLI is still the fastest place to think

The CLI gives me velocity. When I want direct control, fast feedback, and the shortest path from idea to result, the command line is still hard to beat.

  • Inspecting a codebase quickly
  • Running focused checks
  • Moving without UI friction
  • Staying close to the actual system

MCP becomes valuable when structure matters

MCP starts to matter when I want structure instead of improvisation. It forces clearer boundaries between tools, capabilities, and context, which creates a better environment for reliability.

Once the work moves beyond experimentation, repeatability, clearer contracts, and safer handoffs between components matter much more than novelty.

Agents help when coordination becomes the bottleneck

I do not want an agent for everything. I want one when a task is broad enough that micromanagement becomes the expensive part: when context matters, when a problem has multiple steps, and when delegation creates more leverage than driving every action manually.

The real sweet spot is composition

For me, the strongest setup looks like this: the CLI gives speed, MCP gives structure, and agents give leverage.

That combination feels much better than committing fully to any one layer. The CLI keeps me hands-on. MCP makes tool use reliable. Agents reduce coordination overhead on larger tasks.

The mature version of AI work is systems design

The real challenge is not getting a model to output something impressive. The challenge is designing a workflow where intent stays clear, context does not collapse, failures are understandable, output remains reviewable, and humans stay meaningfully in control.

That is why I think the future of working with AI is less about one magical interface and more about composing the right system around it.
